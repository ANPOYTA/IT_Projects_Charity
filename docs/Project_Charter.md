# Project Charter (Хартія проєкту)

\*\*Назва проєкту:** Веб-платформа для благодійних/волонтерських зборів [cite: 39]
\*\*Керівник проєкту (РМ):** Анна Пойта [cite: 40]

## 1. Цілі проєкту

\Розробка та впровадження веб-платформи для вирішення проблем у сфері благодійності в Україні[cite: 43]:

- \*\*Централізація зборів:\*\* Єдиний зручний простір для донорів[cite: 44].
- \*\*Підвищення довіри:\*\* Прозора система верифікації волонтерів та фондів[cite: 45].
- \*\*Забезпечення прозорості:\*\* Функціонал для надійної звітності щодо витрат[cite: 46].
- \*\*Спрощення донату:\*\* Зручний UI/UX та інтегровані платіжні віджети[cite: 47].

## 2. Критерії успіху

- \Стабільна робота клієнтської (FE) та серверної (BE) частин[cite: 50].
- \Успішна інтеграція з API банків (наприклад, підключення "банок")[cite: 51].
- \Проходження тестування безпеки (відсутність критичних вразливостей)[cite: 52].
- \Стійкість системи до DDoS-атак[cite: 53, 54].
- \Регулярне оновлення проєктної документації (WBS, Гант, Risk Log)[cite: 55].

## 3. Високорівневі ризики

- \*\*Ризики безпеки (Критичні):\*\* Кібератаки, злам системи або пошук вразливостей[cite: 59].
- \*\*Технічні ризики:\*\* Баги, що призводять до втрати донатів або помилок у статистиці[cite: 60].
- \*\*Ризики інтеграції:\*\* Затримки при підключенні платіжних шлюзів через API[cite: 61].
- \*\*Організаційні ризики:\*\* Відхилення від таймінгу через складність логіки бази даних чи інтерфейсу[cite: 62].

## 4. Ключові зацікавлені сторони

- \*\*Донори:\*\* Користувачі, що здійснюють внески[cite: 67].
- \*\*Волонтери / Благодійні фонди:\*\* Створюють збори та надають звітність[cite: 68].
- \*\*Банки та платіжні системи:\*\* Партнери для забезпечення транзакцій[cite: 69].
- \*\*Проєктна команда:\*\* Анна Пойта, Маргарита Панченко, Анна Міллер[cite: 70].

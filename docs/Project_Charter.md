# Project Charter (Хартія проєкту)

[cite_start]**Назва проєкту:** Веб-платформа для благодійних/волонтерських зборів [cite: 39]
[cite_start]**Керівник проєкту (РМ):** Анна Пойта [cite: 40]

## 1. Цілі проєкту

[cite_start]Розробка та впровадження веб-платформи для вирішення проблем у сфері благодійності в Україні[cite: 43]:

- [cite_start]**Централізація зборів:** Єдиний зручний простір для донорів[cite: 44].
- [cite_start]**Підвищення довіри:** Прозора система верифікації волонтерів та фондів[cite: 45].
- [cite_start]**Забезпечення прозорості:** Функціонал для надійної звітності щодо витрат[cite: 46].
- [cite_start]**Спрощення донату:** Зручний UI/UX та інтегровані платіжні віджети[cite: 47].

## 2. Критерії успіху

- [cite_start]Стабільна робота клієнтської (FE) та серверної (BE) частин[cite: 50].
- [cite_start]Успішна інтеграція з API банків (наприклад, підключення "банок")[cite: 51].
- [cite_start]Проходження тестування безпеки (відсутність критичних вразливостей)[cite: 52].
- [cite_start]Стійкість системи до DDoS-атак[cite: 53, 54].
- [cite_start]Регулярне оновлення проєктної документації (WBS, Гант, Risk Log)[cite: 55].

## 3. Високорівневі ризики

- [cite_start]**Ризики безпеки (Критичні):** Кібератаки, злам системи або пошук вразливостей[cite: 59].
- [cite_start]**Технічні ризики:** Баги, що призводять до втрати донатів або помилок у статистиці[cite: 60].
- [cite_start]**Ризики інтеграції:** Затримки при підключенні платіжних шлюзів через API[cite: 61].
- [cite_start]**Організаційні ризики:** Відхилення від таймінгу через складність логіки бази даних чи інтерфейсу[cite: 62].

## 4. Ключові зацікавлені сторони

- [cite_start]**Донори:** Користувачі, що здійснюють внески[cite: 67].
- [cite_start]**Волонтери / Благодійні фонди:** Створюють збори та надають звітність[cite: 68].
- [cite_start]**Банки та платіжні системи:** Партнери для забезпечення транзакцій[cite: 69].
- [cite_start]**Проєктна команда:** Анна Пойта, Маргарита Панченко, Анна Міллер[cite: 70].

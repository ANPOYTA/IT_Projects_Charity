# План управління ризиками

**Проєкт:** Веб-платформа для благодійних / волонтерських зборів  
**Мета:** Визначення підходу до виявлення, оцінки та реагування на ризики, пов'язані з фінансовими транзакціями та персональними даними

## 1. Методологія оцінки

Кожен ризик оцінюється за формулою: **Рейтинг = Ймовірність (P) × Вплив (I)**.

| Рівень ризику    | Оцінка (P×I) | Дія                       |
| :--------------- | :----------: | :------------------------ |
| 🔴 **Критичний** |    12-25     | Негайне реагування        |
| 🟠 **Високий**   |     8-11     | Планові заходи реагування |
| 🟡 **Середній**  |     5-7      | Моніторинг та мітігація   |
| 🟢 **Низький**   |     1-4      | Прийняти / Спостерігати   |

## 2. Реєстр ризиків

| ID       | Назва ризику             |  Рівень   | Власник      | Превентивні заходи                      |
| :------- | :----------------------- | :-------: | :----------- | :-------------------------------------- |
| **R-01** | DDoS-атака на платформу  | 🔴 Крит.  | Анна М.      | WAF + CDN (Cloudflare), rate limiting   |
| **R-02** | Витік персональних даних | 🟠 Висок. | Анна М.      | Шифрування AES-256, RBAC, пентест       |
| **R-04** | Збій банківського API    | 🔴 Крит.  | Маргарита П. | Retry-механізм, резервний шлюз (Stripe) |
| **R-05** | Помилки обробки платежів | 🟠 Висок. | Маргарита П. | Idempotency keys, щоденна звірка        |
| **R-08** | Перевищення термінів     | 🟠 Висок. | Анна П.      | Sprint review, буфер часу 15%           |
| **R-10** | Низька залученість (UX)  | 🟡 Серед. | Анна П.      | UX-дослідження, A/B тести               |

## 4. Пріоритетні дії до запуску

1. **R-01:** Впровадити Cloudflare WAF до початку бета-тесту.
2. **R-02:** Провести пентест до продакшн-релізу.
3. **R-04/05:** Налаштувати резервний шлюз та автозвірку транзакцій.
4. Завершити compliance-аудит (GDPR/PCI DSS).

## 5. Відповідальність та моніторинг

- **Анна Пойта (PM):** Щотижневий перегляд (R-08, R-10).
- **Маргарита Панченко (Backend):** Щоденний моніторинг логів та API (R-04, R-05).
- **Анна Міллер (Security):** Аудит безпеки після кожного релізу (R-01, R-02).
